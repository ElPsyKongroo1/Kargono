11 // This namespace provides the main classes that allow the Event Pipeline
12 // to work correctly. The intended use of classes in this namespace
13 // is:
14 // 1. Create an OnEvent(Event) function in the main application layer that
15 // links to other OnEvent(Event) functions depending on the particular scenerio.
16 // (These linkages constitute the event pipeline)
17 // 2. Provide this main OnEvent(Event) function to an event generating section
18 // of the engine such as the Windowing Library (GLFW currently) that can receive
19 // input events such as KeyPressed events.
20 // 3. Add Dispatchers throughout the OnEvent(Event) function pipeline that link
21 // to local functions that handle events. The dispatchers check if the event type
22 // of the currently running event matches the dispatcher's type.
23 // 4. Implement logic to handle events in different scenerios using the m_Handled
24 // member of the event class.

117  //============================================================
118  // This class is a supporting functionality class for the event class. This

142  // Execute and Handle Event
143  //==============================
144  // This function provides a method for ensuring the event initialized with
145  // m_Event has the same EventType as the template type T. Once the
146  // event types match, the provided function is called and the event is
147  // set to handled. This allows for an easy API to externally handle
148  // different scenerios involving event management inside the event
149  // pipeline.
150  template<typename T>
151  bool Dispatch(EventFn<T> func)
152  {
153  if (m_Event.GetEventType() == T::GetStaticType())
154  {

156  return true;
157  }
158  return false;
159  }
160  private:
161  // m_Event holds a reference to the currently managed event inside the
162  // event pipeline. This event is stored internally so that it does
163  // not need to be provided to the Dispatch call repeatadly.
164  Event& m_Event;
165  };
