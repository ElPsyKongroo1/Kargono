43  // Metadata Struct
44  //==============================
45  // This is the generic metadata struct that is directly held inside an asset.
46  // This struct holds generic information about an asset that is universally
47  // needed. The IntermediateLocation provides a relative path from the selected
48  // project's asset directory to the intermediate. As an aside, an intermediate
49  // is file that has been preprocessed and is directly ready to be loaded into
50  // the engine and used without the use of external tools. The checksum is generated
51  // differently based on the file type, but it uniquely identifies the file itself.
52  // The checksum is generally used to ensure duplicate files of the same type are
53  // not being stored. The AssetType allows an easy method to identify the specific
54  // asset type in a generic sense. The SpecificFileData holds a reference to data
55  // that is only needed by this specific type of asset. Ex: A texture might need
56  // to know its width and height, while an audio file might need to know its sample
57  // rate.
58  struct Metadata
59  {
60  public:
61  std::filesystem::path FileLocation;
62  std::filesystem::path IntermediateLocation;
63  std::string CheckSum;
64  Assets::AssetType Type = Assets::AssetType::None;
65  Ref<void> SpecificFileData { nullptr };
66  public:
67  template <typename T>
68  T* GetSpecificMetaData()
69  {
70  return static_cast<T*>(SpecificFileData.get());
71  }
72  };

120  //==============================
121  // Font MetaData Struct
122  //==============================
123  // This metadata struct holds the information needed to correctly load a selected font
124  // from its intermediate(binary) form. Fonts are originally loaded from a .ttf file which
125  // holds most of this metadata itself. When the font is converted into an intermediate form,
126  // the information that defines each character is saved in the Characters vector. This
127  // information includes the character's height, width, etc... The BufferLocations describe
128  // the exact location in the intermediate where each character image is stored. The index of the
129  // Characters and BufferLocations vectors correspond with each other. The Character object
130  // holds enough information to get the size of the image inside of the intermediate.
131  // The AverageWidth and AverageHeight fields are generated when loading the .ttf file and are
132  // used for text positioning in the user interface. The InitialFileLocation field is for
133  // debugging.
134  struct FontMetaData
135  {
136  float AtlasWidth;
137  float AtlasHeight;
138  float LineHeight{};
139  std::vector<std::pair<unsigned char, RuntimeUI::Character>> Characters{};
140  };
