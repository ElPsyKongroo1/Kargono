40  // Metadata Struct
41  //==============================
42  // This is the generic metadata struct that is directly held inside an asset.
43  // This struct holds generic information about an asset that is universally
44  // needed. The IntermediateLocation provides a relative path from the selected
45  // project's asset directory to the intermediate. As an aside, an intermediate
46  // is file that has been preprocessed and is directly ready to be loaded into
47  // the engine and used without the use of external tools. The checksum is generated
48  // differently based on the file type, but it uniquely identifies the file itself.
49  // The checksum is generally used to ensure duplicate files of the same type are
50  // not being stored. The AssetType allows an easy method to identify the specific
51  // asset type in a generic sense. The SpecificFileData holds a reference to data
52  // that is only needed by this specific type of asset. Ex: A texture might need
53  // to know its width and height, while an audio file might need to know its sample
54  // rate.
55  struct Metadata
56  {
57  public:
58  std::filesystem::path IntermediateLocation;
59  std::string CheckSum;
60  Assets::AssetType Type = Assets::AssetType::None;
61  Ref<void> SpecificFileData = nullptr;
62  public:
63  template <typename T>
64  T* GetSpecificFileData()
65  {
66  return static_cast<T*>(SpecificFileData.get());
67  }
68  };

104  // Shader MetaData Struct
105  //==============================
106  // This metadata struct holds information to correctly interpret the shader binary when creating the final Shader object
107  // in the selected rendering API. The ShaderSpec field holds the specification that was originally used to generate
108  // the shader through the shader builder. The UniformList holds the list of uniforms needed for this shader to
109  // run correctly. The InputBufferLayout is generated when a shader is created in the shader builder (the uniform list too)
110  // and holds the format of per vertex input. Ex: vec3(vertex position), vec3(world position), vec4(color), etc...
111  struct ShaderMetaData
112  {
113  ShaderSpecification ShaderSpec{};
114  UniformBufferList UniformList{};
115  InputBufferLayout InputLayout{};
116  };

118  //==============================
119  // Font MetaData Struct
120  //==============================
121  // This metadata struct holds the information needed to correctly load a selected font
122  // from its intermediate(binary) form. Fonts are originally loaded from a .ttf file which
123  // holds most of this metadata itself. When the font is converted into an intermediate form,
124  // the information that defines each character is saved in the Characters vector. This
125  // information includes the character's height, width, etc... The BufferLocations describe
126  // the exact location in the intermediate where each character image is stored. The index of the
127  // Characters and BufferLocations vectors correspond with each other. The Character object
128  // holds enough information to get the size of the image inside of the intermediate.
129  // The AverageWidth and AverageHeight fields are generated when loading the .ttf file and are
130  // used for text positioning in the user interface. The InitialFileLocation field is for
131  // debugging.
132  struct FontMetaData
133  {
134  float AtlasWidth;
135  float AtlasHeight;
136  float LineHeight{};
137  std::vector<std::pair<unsigned char, RuntimeUI::Character>> Characters{};
138  std::filesystem::path InitialFileLocation;
139  };
